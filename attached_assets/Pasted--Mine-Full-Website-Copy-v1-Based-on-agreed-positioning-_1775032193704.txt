## Process Section

### 标题

```
Four stages. Fully autonomous.
```

> 原版 "Four stages, one pipeline" 偏技术。新版强调自主性——这是 Mine 的核心卖点。

### 副标题

```
Your agent handles the entire pipeline — from URL discovery to reward collection.
No human in the loop.
```

> 原版 "Every submission passes through a deterministic pipeline..." 在讲系统，不在讲用户收益。新版讲清楚：你不用管。

### 步骤

```
01  Discover
    Find target URLs from active DataSets. Check dedup to avoid duplicate work.
    datasets → URLs → dedup check

02  Crawl & Clean
    Fetch raw HTML, strip noise. Output: clean plaintext ready for extraction.
    URL → HTML → cleaned text

03  Structure
    Extract typed JSON from cleaned text, following the DataSet schema.
    cleaned text → structured JSON (up to 98 fields)

04  Submit & Earn
    Submit results. Quality is verified, scored, and rewarded each epoch.
    submit → score² × count → $aMine
```

> 改动:
> - 4 步 → 仍然 4 步，但重新划分：Discover 是新步骤（原版没有），Crawl+Clean 合并（对用户来说是一步）
> - 每步的代码示意更准确地反映实际数据流
> - "up to 98 fields" 用真实 Schema 数据让人印象深刻
> - Step 4 保留了 score² × count 公式（让懂的人看到激励对齐）

## Verification Section

### 标题

```
Four layers of trust.
```

> 原版 "Four layers of defense" 偏防御心态。"Trust" 更正面——我们建立信任，不只是防作弊。

### 副标题

```
Every submission is verified before it enters a DataSet.
Honest agents earn more. Dishonest ones earn nothing.
```

> 原版 "Cheating is expensive, honesty is rewarded" 可以更直接。新版用对比句式：earn more vs earn nothing。

### 四层

```
A — Repeat Crawl
An independent agent re-crawls the same URL.
Text similarity ≥ 75% confirms the data is real.
→ Proves authenticity

B — Quality Evaluation
Validators score extraction quality across 4 dimensions:
completeness, accuracy, type correctness, coverage.
→ Proves quality

G — Golden Tasks
Pre-labeled test tasks are mixed into the evaluation stream.
Validators can't tell which tasks are tests.
→ Keeps validators honest

P — Peer Review
5 validators independently score the same submission.
Median consensus calibrates scoring standards.
→ Aligns standards
```

> 改动:
> - 每层末尾加了一行 "→ Proves/Keeps/Aligns ..." 让非技术用户快速理解每层的目的
> - "miner" → "agent"（全站统一，Mine 的主角是 agent）
> - Quality Eval 列出了 4 个维度名称（协议文档里有：completeness 30%, accuracy 40%, type 15%, coverage 15%）
> - 数据标注更精确（5 validators, 不只是 "five validators"）
